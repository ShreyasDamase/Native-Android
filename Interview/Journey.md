So that’s how I transitioned into being a React Native developer."

 why do you like RN ?
 Sure! One of the main things I really like about React Native is its cross-platform capability. You write one codebase, and it runs on both Android and iOS. That saves a ton of time and effort. Another big plus is that it’s all JavaScript-based, so you don’t have to learn Kotlin or Swift. You can just stick to JavaScript, and the underlying architecture—whether it's Fabric or the new Turbo modules—takes care of the platform-specific stuff for you.

I also love how quickly you can build apps with React Native. Compared to native development, it’s usually faster to get something up and running. Plus, things like dependency management are simpler. You just run `yarn install`, add your package, and you’re good to go.

Hot reloading is another favorite feature. As soon as you make changes, you see them instantly without having to recompile the entire app, which is super convenient. While other platforms have something similar, React Native’s hot reloading is really effective.

And lastly, the React Native team is always improving things. They’re active on Discord, they’re continuously resolving performance issues, and they keep the architecture up-to-date. Sure, sometimes you have to wait a bit for all the packages to catch up with new versions, but overall, they’re constantly making it better.

So those are the big things I really enjoy about working with React Native."

HOW TO solve BUGS IN RN APPROACH ?


"Sure! If I run into a bug or a tricky problem while coding, I generally follow a few steps. First, I try to solve it on my own with a bit of trial and error and by checking the documentation to make sure I haven’t missed anything obvious.

If that doesn’t work, I usually head to Stack Overflow or community forums to see if someone else has encountered the same issue. I’ll also jump into the React Native Discord channels because the community there is super active and helpful.

If I’m still stuck, I might use tools like ChatGPT or post on Reddit tagging the React Native teams to get more insights. And in rare cases, if it’s something really challenging and critical, I’ll directly reach out to core maintainers or well-known community members for guidance.

In fact, I’ve done that before and got a quick reply that helped me understand the difference in how Expo’s router handles params versus React Navigation, which cleared up my confusion. So that’s pretty much my approach: start by debugging myself, lean on the community, and use all available resources until I find a solution."
