# 🕵️ Git for Unity
### Version Control, Meta Files, Git LFS & Merge Conflict Mitigation

Using Git with Unity requires specific configurations to manage large binary files, track metadata connections, and resolve merge conflicts in serialized YAML scenes and prefabs.

---

## 1. The Critical Role of Meta Files (`.meta`)

For every asset or folder you add to the `Assets/` directory, Unity creates a corresponding `.meta` file (e.g. `Player.cs` creates `Player.cs.meta`).

*   **GUID Reference Mapping**: Inside the `.meta` file is a unique GUID (Globally Unique Identifier). When you link a script to a GameObject or drag a sprite onto a component, Unity saves the link to the asset's GUID, **not the filename**.
*   **The Golden Rule of Unity Git**:
    > [!IMPORTANT]
    > **Always commit `.meta` files along with their matching asset.** 
    > If you commit an asset but forget its `.meta` file, other developers pulling your branch will generate a new local GUID. This breaks all connections to that asset inside scenes and prefabs, resulting in "Missing Script" or "Missing Sprite" errors.
*   **Editor Check**: Ensure **Version Control Mode** is set to `Visible Meta Files` in `Project Settings -> Version Control` (default in modern Unity).

## 3. Git LFS (Large File Storage) & `.gitattributes`

Git stores history by saving text diffs. For large binary files (sprites, textures, audio, 3D models), Git cannot calculate diffs and instead copies the entire file for every edit, which quickly bloats your repository size.

*   **Git LFS**: Replaces large binary assets with tiny text pointers in your Git repository while storing the actual assets on an external server.
*   **`.gitattributes` configuration**: Create a file named `.gitattributes` at the root of your repository to track binary structures via LFS and force Unity files to use consistent line endings:

```
# ======================== #
# Git LFS Binary Tracking  #
# ======================== #
*.psd filter=lfs diff=lfs merge=lfs -text
*.png filter=lfs diff=lfs merge=lfs -text
*.jpg filter=lfs diff=lfs merge=lfs -text
*.wav filter=lfs diff=lfs merge=lfs -text
*.mp3 filter=lfs diff=lfs merge=lfs -text
*.ogg filter=lfs diff=lfs merge=lfs -text
*.fbx filter=lfs diff=lfs merge=lfs -text
*.blend filter=lfs diff=lfs merge=lfs -text
*.pdf filter=lfs diff=lfs merge=lfs -text
*.zip filter=lfs diff=lfs merge=lfs -text
*.tga filter=lfs diff=lfs merge=lfs -text

# ======================== #
# Unity YAML Line Endings  #
# ======================== #
*.unity text eol=lf
*.prefab text eol=lf
*.asset text eol=lf
*.meta text eol=lf
```

### Basic Git LFS Commands
```bash
# 1. Initialize LFS on your machine
git lfs install

# 2. Add files to track manually (if not covered in .gitattributes)
git lfs track "*.mp4"

# 3. View tracked files
git lfs status
```

## 4. Resolving Scene & Prefab Merge Conflicts

Unity Scenes (`.unity`) and Prefabs (`.prefab`) are serialized as complex YAML text blocks. Resolving merge conflicts inside these files manually is difficult and can corrupt them.

### Best Practices to Prevent Conflicts
1.  **Use Prefabs for Level Design**: Instead of placing level elements directly in a main scene, package elements (decorations, player configs, obstacles) into separate Prefabs. Multiple developers can edit separate prefabs without causing conflict updates to the parent scene.
2.  **Modular Additive Scenes**: (See Prefabs & Scenes). Divide scenes into `Level1_Art`, `Level1_Physics`, and `Level1_Audio`. Developers open only their target scene.
3.  **Use File Locking (Git LFS Locks)**: Communicate with team members when you are working on a shared scene, or lock the file:
    ```bash
    git lfs lock Assets/Scenes/MainLevel.unity
    # Release lock when done
    git lfs unlock Assets/Scenes/MainLevel.unity
    ```

### Configuring Unity SmartMerge (UnityYAMLMerge)
Unity includes a built-in merge tool called `UnityYAMLMerge` that can resolve conflicts in scene and prefab files automatically.

To configure Git to use Unity's SmartMerge, run the following commands (or add them to your user `.gitconfig` file):

```bash
# Configure UnityYAMLMerge path for Git (Example path for macOS Unity installation)
git config --global merge.unityyamlmerge.name "Unity SmartMerge"
git config --global merge.unityyamlmerge.driver "'/Applications/Unity/Hub/Editor/6000.0.35f1/Unity.app/Contents/Tools/UnityYAMLMerge' merge -p --force %O %B %A %R"

# Set up Git to use this driver for Unity YAML files (matches our .gitattributes settings)
git config --global merge.unity.driver true
```
