## 1. ACTIVITY & FRAGMENT LIFECYCLE

> Core of every Android interview. Expect rotation, back-stack, and low-memory scenarios.

### Activity Lifecycle

- [ ] 🔴 List all Activity lifecycle callbacks in order for: first launch, Home button, screen rotation, finishing
- [ ] 🔴 What is the difference between `onPause()` and `onStop()`?
- [ ] 🟡 When is `onDestroy()` NOT called?
- [ ] 🟡 What happens to the Activity when the system is **low on memory**? Which callbacks fire?
- [ ] 🟡 What are the 4 Activity **launch modes**? (`standard`, `singleTop`, `singleTask`, `singleInstance`) — explain with back-stack behaviour
- [ ] 🟢 What is the difference between `AppCompatActivity` and `ComponentActivity`?
- [ ] 🔴 Why must Activities be declared in `AndroidManifest.xml`? What would happen if they weren't?

### Fragment Lifecycle

- [ ] 🔴 List the complete Fragment lifecycle — all callbacks in order from creation to destruction
- [ ] 🔴 At what point is the Fragment's **view** created and destroyed _separately_ from the Fragment itself? (`onCreateView` vs `onDestroyView` vs `onDestroy`)
- [ ] 🟡 What is the difference between `onAttach()`, `onCreateView()`, and `onViewCreated()`? Where should you initialise views and why?
- [ ] 🟡 What happens to a Fragment when the host Activity is **stopped**?
- [ ] 🟡 What is `FragmentManager` and `FragmentTransaction`? List 2 common pitfalls (e.g., committing after `onSaveInstanceState`)
- [ ] 🟡 What is `setRetainInstance()` (deprecated)? What problem did it solve and what **replaced** it?
- [ ] 🟢 What happens to a Fragment's ViewModel when the user switches tabs and comes back?

## 2. VIEWMODEL & STATE MANAGEMENT

> Every company asks about ViewModel internals. "How does it survive rotation?" is almost guaranteed.

### ViewModel Internals

- [ ] 🔴 How does ViewModel survive a **configuration change** (screen rotation)? Explain step-by-step what the system does
- [ ] 🔴 What is `ViewModelStore`? Who owns it?
- [ ] 🔴 How does `ViewModelProvider` find an existing ViewModel vs create a new one?
- [ ] 🔴 When is the ViewModel actually **destroyed** (not just recreated)?

### Scopes

- [ ] 🔴 What is the difference between `viewModelScope` and `lifecycleScope`? When would you use each?
- [ ] 🟡 What happens when an Activity with an active `viewModelScope` coroutine is **finished** (not rotated)?

### State Holders

- [ ] 🔴 What is the difference between `StateFlow`, `SharedFlow`, and `LiveData`? When would you choose each?
- [ ] 🟡 What is `StateFlow`'s replay behaviour vs `SharedFlow`'s `replay` parameter?
- [ ] 🟡 Why would you prefer `StateFlow` over `LiveData` in a new project?

### AsyncTask (Legacy but still asked at Paytm)

- [ ] 🟢 What is the **AsyncTask rotation problem**? How do you solve it in modern Android?

### Data Persistence Across Rotation

- [ ] 🟡 Your stock list was loaded from the API before rotation. After rotation, does it make another API call? Why or why not? (trace through Repository + ViewModel)
- [ ] 🟢 What is `onSaveInstanceState` vs ViewModel for preserving UI state? When do you use each?

## 3. COROUTINES & FLOW

> Most heavily tested Kotlin topic. Rapido, PhonePe, InCred, Zomato all go deep here.

### Coroutine Basics

- [ ] 🔴 What are coroutines? How do they differ from threads?
- [ ] 🔴 What is a `suspend` function? How does suspension work under the hood? What is a `Continuation`?
- [ ] 🔴 What is the difference between `launch` and `async`? When would you use each?
- [ ] 🔴 What is `withContext`? How does it differ from `launch` and `async`?

### Dispatchers

- [ ] 🔴 What is the difference between `Dispatchers.IO`, `Dispatchers.Default`, and `Dispatchers.Main`?
- [ ] 🟡 What is `Dispatchers.Unconfined`? When might it be useful or dangerous?

### Structured Concurrency

- [ ] 🔴 What is **structured concurrency** in Kotlin Coroutines?
- [ ] 🔴 What happens to **child coroutines** if the parent is cancelled?
- [ ] 🔴 What is `SupervisorJob`? When would you use it over a regular `Job`?
- [ ] 🟡 What is `supervisorScope`? How does it differ from `coroutineScope`?

### Exception Handling

- [ ] 🔴 How do you handle exceptions in coroutines? Explain the difference between `CoroutineExceptionHandler`, `try/catch`, and `supervisorScope`
- [ ] 🟡 What potential issue exists with `CoroutineScope(Dispatchers.Main).launch` inside an Activity method? What happens if the Activity is destroyed before the coroutine completes?

### Flow

- [ ] 🔴 What is the difference between a **cold Flow** and a **hot Flow**?
- [ ] 🔴 What is the difference between `Flow`, `StateFlow`, and `SharedFlow`?
- [ ] 🟡 What is `flowOn()`? How does it switch the context of a cold flow?
- [ ] 🟡 What is `conflate()`? When would you use it?
- [ ] 🟡 How do you implement **debounced search** using Flow? (hint: `debounce` + `distinctUntilChanged`)
- [ ] 🟢 What is `collectLatest` vs `collect`? What problem does `collectLatest` solve?

### Cancellation

- [ ] 🟡 What is `job.cancel()` followed by `job.join()`? What does each do and why use both?
- [ ] 🟡 What happens to a coroutine if you remove `job.join()` after cancelling it?

## 8. KOTLIN LANGUAGE

> Rapido and InCred go deepest on Kotlin. Expect internals, not just syntax.

### Classes & Types

- [ ] 🔴 What is the difference between a regular `class`, `data class`, and `object` in Kotlin?
- [ ] 🔴 What does the compiler **auto-generate** for a `data class`? (equals, hashCode, toString, copy, componentN)
- [ ] 🟡 Can a data class be empty (no properties)? What is the minimum requirement?
- [ ] 🔴 What is the difference between `sealed class` and `enum class`? When would you choose a sealed class?
- [ ] 🟡 What is a `companion object`? How does it differ from `object`?

### Null Safety

- [ ] 🔴 How does Kotlin ensure null safety?
- [ ] 🔴 What is the difference between `?.`, `!!`, `?:`, and `let`? When is `!!` dangerous?
- [ ] 🟡 What are **nullable types** (`Int?`) and where are they stored — stack or heap?

### Functions

- [ ] 🔴 What are **extension functions**? How do they work internally at the bytecode level?
- [ ] 🟡 What are **inline functions**? When and why would you use them? (avoids lambda allocation)
- [ ] 🟡 What are **higher-order functions** and **closures**?
- [ ] 🟡 What are **lambda functions**? Give 3 real use cases from an Android app
- [ ] 🟢 What is a **trailing lambda**?

### Delegation & Properties

- [ ] 🔴 What is **delegation** in Kotlin? Explain `by lazy` vs `lateinit`
- [ ] 🔴 What happens if you access a `lateinit` variable before initialising it?
- [ ] 🟡 What is the difference between `val` and `const val`? Where is each stored in memory?

### OOP

- [ ] 🟡 Explain `open`, `abstract`, `interface`, and `sealed` in Kotlin — when to use each
- [ ] 🟡 What are visibility modifiers (`private`, `protected`, `internal`, `public`)? How does `internal` differ from `public`?

### Scope Functions

- [ ] 🟡 Explain all 5 scope functions: `let`, `run`, `with`, `apply`, `also` — receiver (`this`/`it`), return value, best use case for each

### Enums

- [ ] 🟢 What are **enums with properties** in Kotlin? Write an enum with a property and a method

### KMM (InCred specific)

- [ ] 🟢 What does KMM share between Android and iOS, and what does it **not** share?
- [ ] 🟢 What is the `expect` / `actual` pattern in KMM?

## 10. MEMORY, PERFORMANCE & SECURITY

> PhonePe and InCred go deepest here. Paytm tests GC internals.

### Memory Leaks

- [ ] 🔴 What is the difference between a **memory leak** and an **OOM error**?
- [ ] 🔴 List 3 common causes of memory leaks in Android (e.g., Activity reference in singleton, static context, unregistered listener)
- [ ] 🟡 A ViewModel holds a reference to a Fragment's View. Why is this a memory leak and how do you fix it?
- [ ] 🟡 How do you detect memory leaks? Name the tools (LeakCanary, Android Profiler)

### Garbage Collection

- [ ] 🟡 How does Android's garbage collector work? What is the **generational GC model**?
- [ ] 🟡 How do you avoid triggering excessive GC? (avoid allocations in `onDraw`, loops)

### Performance Debugging

- [ ] 🔴 Your app shows **ANR** (Application Not Responding) dialogs. Walk through your entire debugging process
- [ ] 🟡 Your RecyclerView **scrolls janky** (drops frames). Walk through your entire debugging process
- [ ] 🟡 What is `StrictMode`? When would you use it?
- [ ] 🟢 Name 5 concrete techniques to optimise an Android app's performance

### Security

- [ ] 🟡 What is a **Man-in-the-Middle (MITM) attack** on Android? How do you prevent it? (certificate pinning)
- [ ] 🟡 How do you implement **certificate pinning** with OkHttp?
- [ ] 🟡 How do you store **sensitive data** (API keys, tokens) securely? (EncryptedSharedPreferences, Android Keystore)
- [ ] 🟢 A phone is rooted. An attacker decompiles your APK and recompiles it. How do you detect this server-side? (SafetyNet / Play Integrity API)

## STUDY ORDER RECOMMENDATION

If you are starting fresh, study in this order:

1. **Kotlin Language** — foundation for everything
2. **Activity & Fragment Lifecycle** — asked in every single interview
3. **ViewModel & State Management** — guaranteed at mid-senior level
4. **Coroutines & Flow** — guaranteed at all levels now
5. **Architecture Patterns** — MVVM + Repository is the baseline
6. **RecyclerView & UI Rendering** — comes up in every practical round
7. **Services & Background Work** — asked at most companies
8. **Dependency Injection** — Hilt is now standard
9. **Jetpack Compose** — growing fast; must-know for PhonePe and InCred
10. **Memory, Performance & Security** — separates mid from senior
11. **Networking & IPC** — system design adjacent topics
12. **Data Persistence** — comes up less but tested at PhonePe, Paytm
